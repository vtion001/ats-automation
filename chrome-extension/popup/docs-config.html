Client Configuration

Each client has its own configuration file located in clients/[client_name]/config.json.

Configuration Structure
{
  "client_name": "flyland",
  "display_name": "Flyland Recovery",
  "systems": {
    "ctm": { "enabled": true },
    "salesforce": { "enabled": true }
  },
  "automations": {
    "automation_name": {
      "enabled": true,
      "priority": "HIGH",
      "phase": 1,
      "delay_ms": 500,
      "timeout_ms": 5000
    }
  },
  "logging": {
    "level": "INFO",
    "save_to_file": true
  },
  "nlp": {
    "enabled": true,
    "confidence_threshold": 0.7
  }
}
System Configuration
Setting Type Description
enabled boolean Enable/disable the system
url_pattern string URL pattern to match (e.g., *.calltrackingmetrics.co)
filters object Additional filters (e.g., record_type, owner)
Automation Settings
Setting Type Description
enabled boolean Enable/disable this automation
priority string HIGH, MEDIUM, or LOW
phase number Implementation phase (1, 2, or 3)
delay_ms number Delay before triggering (milliseconds)
timeout_ms number Timeout for operation (milliseconds)
DOM Profiles

DOM profiles define how to interact with different web pages. Located in clients/[client]/dom_profiles/[system].json.

Profile Structure

{
  "system": "ctm",
  "selectors": {
    "call_notification": ".caller-id, .call-info",
    "phone_number": "#phone-field, .phone-input"
  },
  "event_selectors": {
    "incoming_call": ".incoming-call",
    "call_ended": ".call-ended"
  }
}
Templates

Reply and note templates are stored in clients/[client]/templates/.

Reply Templates

{
  "templates": {
    "general_inquiry": [
      "Thank you for reaching out..."
    ],
    "callback_request": [
      "We'll call you shortly..."
    ]
  }
}
Knowledge Base

Qualification criteria and routing rules in clients/[client]/knowledge-base/.

Qualification Criteria

  • hot_lead - Score 70+, immediate transfer to intake
  • warm_lead - Score 40-69, nurture with info
  • cold_lead - Score 0-39, add to follow-up queue
Server Configuration

For multi-agent deployments, configure the server in .env:

ATS_SERVER_URL=http://ags-ai-server.eastus.azurecontainer.io:8000
ATS_API_KEY=your-api-key-here
Note: For local single-user mode, no server configuration is needed. The extension runs standalone.
Field Mappings

Configure field mappings for PDF auto-filling and form automation in config/field-mappings/[client].json.
