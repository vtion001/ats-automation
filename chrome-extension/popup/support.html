Frequently Asked Questions
Q: Why isn't the automation detecting my calls?
Make sure you're logged into CTM and the extension is enabled. Check that the correct client is selected in the popup. If using Salesforce, ensure you're on the correct instance (flyland.my.salesforce.com for Flyland).
Q: The extension shows "Not connected" - what do I do?
This means the local server isn't running. Start the server with python main.py server or check your connection settings in Config.
Q: Can I use this with multiple clients?
Yes! Use the client selector in the popup to switch between clients. Each client has its own configuration, automations, and knowledge base.
Q: Is my data secure?
Yes. The automation works on your existing logged-in sessions - no credentials are stored. All NLP processing happens locally on your machine. No data is transmitted to external servers.
Q: The Salesforce lookup isn't working - what should I check?
1) Verify you're logged into Salesforce in the same browser
2) Check that your Salesforce URL matches the client config
3) For Flyland/TBT - ensure you're using the correct instance (shared SF)
4) Try refreshing the Salesforce page
Q: How do I customize automations for my workflow?
Open Config from the popup menu. You can adjust automation triggers, timeouts, and enable/disable specific features per client.
Troubleshooting

Common Error Codes

  • ERR_CTM_001 - CTM page not detected. Make sure you're on calltrackingmetrics.co
  • ERR_SF_001 - Salesforce session expired. Re-login to Salesforce
  • ERR_SF_002 - No records found for the phone number
  • ERR_DOM_001 - Page elements not found. Page may have updated
  • ERR_TIMEOUT - Operation timed out. Try again or increase timeout in Config
  • ERR_NLP_001 - NLP model failed to load. Check installation
Tip: Many issues can be resolved by refreshing the page and ensuring you're logged into all required systems.
Logs & Diagnostics

To help troubleshoot issues, check the logs:

  • Extension logs: Right-click extension → Popup → Check console
  • Python logs: Check the logs/ folder in the project directory
  • Server logs: Check terminal output when server is running

When reporting an issue, please include:

  • Error code (if any)
  • Client name
  • Browser version
  • Steps to reproduce
System Requirements
  • Browser: Chrome 90+ or Edge 90+
  • OS: Windows 10/11 or macOS
  • Python: 3.10+
  • RAM: 8GB minimum (16GB recommended for NLP)
  • Network: Access to client portals (CTM, SF, etc.)
